# 修复视频带水印问题

## 任务描述
解析出的视频带水印，需要改为提取无水印链接。

## 原因分析
- 之前使用 `dy-downloader` 的 `fetchFromSharePage`，它从分享页提取 `video.play_addr`（带水印）
- f2 工具使用的是 `video.bit_rate[0].play_addr`（无水印）
- 分享页的 `_ROUTER_DATA` 实际包含完整的 `bit_rate` 数据，只是库没有使用

## 待办
- [x] 分析 `fetchFromSharePage` 返回带水印 URL 的原因
- [x] 实现自己的分享页解析，优先从 `bit_rate[0].play_addr` 提取无水印链接
- [x] 验证构建通过

## 修改文件
- `src/lib/parser.ts` - 替换 `fetchFromSharePage`，实现 `fetchSharePageDetail` 和 `extractNoWatermarkUrl`

## 结果
- 不再依赖 `fetchFromSharePage`，自己解析分享页 HTML
- 优先从 `bit_rate[0].play_addr` 提取无水印视频链接
- `bit_rate` 不可用时 fallback 到 `play_addr`
- 前端零影响，数据结构不变
